# Éimí Retro Website – Project Instructions

## Project Vision

Recreate https://eim.ie/ as a functional retro-era website—not just a visual theme, but actual older-web technology that works on vintage devices (PowerPC iMac, PowerBooks, early 2000s hardware). The site combines turn-of-millennium aesthetics with a modern Irish developer identity.

## Brand Identity

**Éimí — "Minimalist Tech Gael"**

The tone: warm, accessible, intelligent, calm. Like an Irish teacher who cares about accessibility and open source.

### Core Values

- Gaeilge-first (Irish before English, equally prominent)
- Accessibility-conscious (WCAG 2.2 AA minimum, AAA where practical)
- Indie developer aesthetic (not corporate, not AI hype)
- Restrained Irish modernism (no tourist clichés, no busy Celtic patterns)
- Breathable, calm design with generous whitespace

### Avoid

- Loud gradients, glassmorphism, over-design
- Busy Celtic patterns, corporate SaaS aesthetics
- Tiny text, thin low-contrast UI
- Aggressive animations, fake "AI futurism"

## Typography

**Body Font:** Atkinson Hyperlegible Next or Inter

- Highly readable, accessibility-oriented
- Minimum 1rem body size
- 1.5–1.7 line-height (never cramped)

**Heading Font:** Fraunces or clean modern serif

- Literary, thoughtful, calm, slightly academic
- Not fantasy Celtic or faux medieval

**Monospace:** JetBrains Mono

- Code blocks, terminal snippets, technical UI

---

## Language Philosophy

**Gaeilge First**

Irish should appear before English, equally prominent.

Example structure:

```
Éimí Mhic an Ridire
Accessibility-focused developer and Gaeilgeoir
```

- Visually distinguish both languages clearly
- Allow proper spacing between language blocks
- Support proper fada rendering everywhere
- Never hide Irish in tiny grey subtitles

---

## Accessibility (Critical)

**Minimum target:** WCAG 2.2 AA (AAA preferred where practical)

### Contrast

- Never low-contrast muted text for important content
- Maintain strong contrast in buttons, links, hover states, disabled states, forms

### Links

- Always visibly identifiable (preferably underlined)
- Don't rely solely on color

### Focus States

- Obvious, keyboard-visible, high contrast
- 2–3px visible focus ring using brand lavender/periwinkle

### Motion

- Subtle animations only
- Respect `prefers-reduced-motion`
- Never block interaction

### Touch Targets

- Minimum 44x44px interactive elements
- Especially important for mobile and motor accessibility

### Language Markup

- Use proper `lang="ga"` and `lang="en"` attributes
- Improves screen reader pronunciation, indexing, translation tooling

---

## Layout & CSS

### General Feel

- Structured, breathable, soft-edged, intentionally calm
- Card layouts with border-led separation
- Restrained shadows, rounded corners (~16px)
- Avoid clutter, edge-to-edge density, floating chaos UI

### Borders & Shadows

- Prefer visible 1–2px borders
- Subtle shadows only where needed
- Not giant blurry shadows or floating glass panels

### Celtic Design Usage

- Sparse, intentional, signature-level detail only
- Good: divider accents, corner motifs, subtle frame decoration
- Bad: tiled knot backgrounds, heavy parchment aesthetics, "Irish pub" visuals
- Irishness should feel lived-in and modern

### CSS Priorities

1. Readability
2. Accessibility
3. Maintainability
4. Semantic naming
5. Graceful responsiveness

### Recommended Values

- Border radius: ~16px
- Transition speed: 150–250ms
- Container width: 1100–1200px max
- Reading width: 65–75ch
- Spacing scale: 8px rhythm

---

## Retro Technology Requirements

Build for compatibility with early-2000s web standards:

- **HTML:** Semantic, lean markup (HTML 4.01 compatible where practical)
- **CSS:** CSS 2.1/CSS 3 where widely supported (avoid modern grid/flexbox complexity if compatibility matters)
- **JavaScript:** Minimal, progressive enhancement (graceful degradation)
- **Performance:** Keep file sizes lean (essential for older bandwidth)
- **Standards compliance:** Focus on web standards, not modern frameworks

Goal: Site should load and function on:

- PowerPC Macs (OS 9/OS X 10.3–10.5)
- Early PowerBooks
- Vintage browsers (Safari 1.x–3.x, iCab, older Opera versions)

## Implementation Principles

1. **Semantic HTML first** – clear structure, accessibility built in
2. **Progressive enhancement** – works without JavaScript
3. **Mobile-first responsive design** – but also test on wide/vintage displays
4. **Generous whitespace** – breathing room between sections
5. **Visible interaction states** – focus, hover, active, disabled all clear
6. **Bilingual by design** – Irish and English equally respected
7. **Inclusive language** – neurodivergent-friendly, plain language
8. **Performance matters** – smaller files, faster loads on older connections

## Quick Reference: UI Components

### Links

- Underlined, color `--brand-500`
- Hover: color `--brand-600`, darker
- Focus: 2–3px ring in `--focus-ring`
- Visited: optional muted color

### Buttons

- Background: `--brand-500`
- Text: white
- Padding: 0.75rem 1.5rem (min 44x44px touch target)
- Border radius: 16px
- Hover: darker shade
- Focus: visible ring

### Cards

- Border: 1px `--border-soft`
- Padding: 1.5rem
- Border radius: 16px
- Subtle shadow optional: `0 2px 8px rgba(0,0,0,0.08)`

### Forms

- Labels: required, clear, `--text-ink`
- Inputs: 1px border `--border-soft`, focus ring on focus
- Validation: use color + icon, not color alone
- Min 44x44px touch targets

### Text Utility Classes

- `.text-muted` – secondary/less important text
- `.language-block` – Irish or English text group
- `.sr-only` – screen reader only

## Questions for Implementation?

Always refer back to:

1. Is this accessible? (WCAG AA minimum)
2. Does this work on retro hardware?
3. Is Irish visible and equally respected?
4. Does this feel calm and intentional?
5. Is the code semantic and maintainable?

When in doubt, choose simplicity, readability, and accessibility.
